1\. System Architecture

본 시스템은 3계층 구조(Three-tier Architecture)를 기반으로 설계되었다.

Presentation Layer (Frontend): 사용자 인터페이스를 담당하며 Web Browser에서 실행된다.

Application Layer (Backend): 비즈니스 로직을 처리하며 Node.js 서버에서 실행된다.

Data Layer (Database): 사용자 및 작업 데이터를 저장하며 SQLite를 사용한다.

시스템 구조는 다음과 같다:

User → Web Browser → Backend API → Database

본 시스템은 다음과 같은 기능적 요구사항을 가진다.

2.1 User Management

사용자 회원가입 기능 제공

사용자 로그인 기능 제공

2.2 Task Management

작업 생성 기능

작업 조회 기능

작업 삭제 기능

2.3 Statistics

작업 완료율 계산

완료/미완료 작업 수 제공



3\. Non-Functional Requirements

본 시스템은 다음과 같은 비기능적 요구사항을 만족해야 한다.

응답 시간: 2초 이내

데이터 일관성 유지

기본 보안 기능 (인증 및 암호화)

사용자 친화적인 인터페이스 제공

시스템 안정성 확보



4\. Database Design

본 시스템의 데이터베이스 구조는 다음과 같다.

4.1 Users Table

Field	Type	Description

id	INTEGER	Primary Key

username	TEXT	사용자 이름

password	TEXT	암호화된 비밀번호

4.2 Tasks Table

Field	Type	Description

id	INTEGER	Primary Key

user\_id	INTEGER	사용자 ID (Foreign Key)

title	TEXT	작업 제목

status	INTEGER	작업 상태 (0: 미완료, 1: 완료)



5\. API Design

본 시스템은 RESTful API 방식으로 설계되었다.

5.1 Authentication API

Method	Endpoint	Description

POST	/auth/register	사용자 등록

POST	/auth/login	사용자 로그인

5.2 Task API

Method	Endpoint	Description

GET	/tasks	작업 목록 조회

POST	/tasks	작업 생성

DELETE	/tasks/	작업 삭제



6\. Sequence Flow

작업 조회 과정은 다음과 같이 수행된다.

사용자가 브라우저에서 요청을 보낸다

프런트엔드는 Backend API로 요청을 전달한다

Backend는 데이터베이스에서 작업 데이터를 조회한다

조회 결과를 프런트엔드로 반환한다

사용자에게 작업 목록이 표시된다



7\. System Design Considerations

본 시스템 설계 시 다음 사항을 고려하였다.

7.1 확장성 (Scalability)

향후 기능 확장을 고려하여 모듈화된 구조로 설계하였다.

7.2 보안성 (Security)

JWT 기반 인증을 적용하여 사용자 정보를 보호한다.

7.3 성능 (Performance)

간단한 데이터 구조를 사용하여 빠른 응답 속도를 확보하였다.

7.4 유지보수성 (Maintainability)

코드 구조를 분리하여 유지보수 및 기능 추가를 용이하게 하였다.
